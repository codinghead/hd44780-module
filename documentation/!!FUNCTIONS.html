lcdifFixNibbleSwap 
lcdifFixNibbleSwap()
For use with an LCDIF that has been opened. During development it was noted that some LCD modules returned the nibbles of data during a "ReadAddress" in the reverse order to that defined in the datasheet or during a "ReadData" command. Calling this function ensures that "ReadAddress" commands work correctly with such LCD module. If, when using the HD44780 Module, you cannot write more than eight characters to the LCD display, you probably have an LCD display with this nibble-swap issue.
